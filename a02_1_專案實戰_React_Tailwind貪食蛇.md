# 專案實戰：使用 Claude Code 打造 React + Tailwind 貪食蛇

這份實戰指南將帶你把前面《快速上手》學到的概念化為現實。我們將完全依賴 Claude Code，從小步迭代開始，完成一個具備完整生命週期的貪食蛇網頁遊戲。

## 為什麼選這個專案？
1. **即時的回饋感**：遊戲有畫面、有互動，每一次 Claude Code 產出的結果都能立刻驗證。
2. **練習小步迭代**：遊戲邏輯（如蛇的移動、吃蘋果、死亡判定）非常適合被切分成小任務。
3. **沒有複雜的後端干擾**：完全在前端瀏覽器執行，讓你專注體驗 Claude Code 的工作流。

---

## 步驟一：初始化專案與設定 CLAUDE.md

首先，請在你的電腦上建立一個空資料夾，並在該目錄下啟動 `claude`：

```bash
mkdir snake-game
cd snake-game
claude
```

進入 Claude Code 後，我們不急著寫程式，先讓它幫我們搭好舞台：

> **你的提示詞：**
> 我想從零建立一個 Vite + React + Tailwind 專案，核心目標是開發網頁版貪食蛇。
> 先告訴我你建議的初始化方式，先不要執行。
> 
> 確認後再幫我自動建立，並且在根目錄生成一份合適的 `CLAUDE.md`，把「不要隨意新增外部套件」、「元件放 src/components」等規範寫進去。

**這個步驟的重點**：讓 Claude 自動執行 `npm create vite@latest` 等繁瑣的終端機指令，並幫你打好專案地基。

---

## 步驟二：繪製網格與實作基礎邏輯

專案與 `CLAUDE.md` 建立完成後，我們進入 Plan Mode 規劃遊戲的核心。

> **你的提示詞：**
> /plan
> 我們來實作貪食蛇的核心邏輯。請幫我規劃：
> 1. 如何用二維陣列或座標系統表示 20x20 的網格？
> 2. 蛇的資料結構該長怎樣？
> 3. 如何監聽鍵盤方向鍵來改變移動方向？
> 
> 請先給出技術方案，不要直接改程式碼。

當 Claude 條列出清晰的思路（例如使用 React 的 `useState` 和 `useEffect` 配合 `setInterval`）後，你再下達執行指令：

> **你的提示詞：**
> 方案看起來很棒，請開始修改程式碼。修改完成後，請確保 `npm run dev` 能正常啟動，讓我看看畫面上有一條會動的蛇。

---

## 步驟三：加入蘋果與分數機制（小步迭代）

確認基本的蛇能移動後，我們開始「小步迭代」加入新功能。

> **你的提示詞：**
> 接下來，幫我加上隨機產生的蘋果。
> 當蛇的頭碰到蘋果時，蛇的長度要增加，並且在畫面上方用 Tailwind 樣式做一個漂亮的計分板。
> 改完之後請用 `/diff` 讓我看看你改了哪裡。

**這個步驟的重點**：不要一次把所有遊戲規則講完。先讓蛇能動，再讓蛇能吃東西，這是確保 AI 不會寫出一團義大利麵程式碼（Spaghetti Code）的秘訣。

---

## 步驟四：實作死亡判定與 Game Over 畫面

> **你的提示詞：**
> 現在我們要處理死亡判定：
> 1. 如果蛇撞到牆壁（超出 20x20 網格）
> 2. 如果蛇咬到自己的身體
> 
> 觸發死亡時，遊戲要暫停，並在畫面正中央彈出一個半透明的 Game Over 視窗，裡面包含最終得分以及一個「重新開始」按鈕。請直接實作。

---

## 步驟五：視覺美化與最後的 Code Review

遊戲已經可以玩了，最後我們讓 Claude 進行程式碼審查與重構，並用 Tailwind 美化它。

> **你的提示詞：**
> /review
> 請從效能與 React 最佳實踐的角度審查我們剛才寫的貪食蛇邏輯。有沒有哪些地方會造成不必要的 re-render？有的話請幫我重構。
> 
> 另外，請發揮你的設計感，用 Tailwind CSS 把遊戲畫面美化成「復古街機」的風格（例如使用螢光綠搭配深黑色背景）。

完成滿意的修改後，使用以下命令收尾：

```bash
/diff
/commit
```
Claude 會自動幫你寫出像 `feat: implement snake collision and retro UI` 這樣的 commit message 並完成提交。
