# 快速入門

歡迎使用 Claude Code！本快速入門指南將在幾分鐘內讓您使用 AI 驅動的程式設計輔助。完成本指南後，您將了解如何使用 Claude Code 完成常見的開發任務。

---

## 開始前的準備

確保您擁有：

- 已開啟的終端機或命令提示字元
  - 如果您之前從未使用過終端機，請查看[終端機指南](https://code.claude.com/docs/terminal-guide)
- 一個可以使用的程式碼專案
- 一個 Claude 訂閱（Pro、Max、Teams 或 Enterprise）、Claude Console 帳號，或透過支援的雲端供應商的存取權限

> 本指南涵蓋終端機 CLI。Claude Code 也可在網頁、桌面應用程式、VS Code 和 JetBrains IDE、Slack 中使用，以及透過 GitHub Actions 和 GitLab 進行 CI/CD。

---

## 步驟 1：安裝 Claude Code

### macOS / Linux / WSL

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### Windows PowerShell

```powershell
irm https://claude.ai/install.ps1 | iex
```

### Windows CMD

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

> **提示**：如果看到 `The token '&&' is not a valid statement separator`，表示您在 PowerShell 而非 CMD；如果看到 `'irm' is not recognized`，表示您在 CMD 而非 PowerShell。PowerShell 提示字元以 `PS C:\` 開頭，CMD 則為 `C:\`。

> Windows 原生安裝需要先安裝 [Git for Windows](https://git-scm.com/downloads/win)。WSL 環境不需要。

> 原生安裝方式會在背景自動更新，確保您使用最新版本。

### Homebrew（macOS）

```bash
brew install --cask claude-code
```

> Homebrew 提供兩個 cask：`claude-code`（穩定版，約晚一週）和 `claude-code@latest`（最新版）。**不會自動更新**，請定期執行 `brew upgrade claude-code`。

### WinGet（Windows）

```powershell
winget install Anthropic.ClaudeCode
```

> **不會自動更新**，請定期執行 `winget upgrade Anthropic.ClaudeCode`。

### Linux 套件管理員

支援 Debian（apt）、Fedora/RHEL（dnf）、Alpine（apk）等發行版，詳見官方文件。

## 步驟 2：登入您的帳號

Claude Code 需要帳號才能使用。啟動互動式會話時系統會提示您登入：

```bash
claude
# 首次使用時系統會提示您登入
```

或在會話內執行：

```bash
/login
```

支援的帳號類型：

- **Claude Pro、Max、Teams 或 Enterprise**（推薦）
- **Claude Console**（具有預付費額度的 API 存取）
- **Amazon Bedrock、Google Vertex AI 或 Microsoft Foundry**（企業雲端供應商）

登入後，憑證會被儲存，下次無需再次登入。若要切換帳號，請使用 `/login` 命令。

---

## 步驟 3：啟動您的第一個會話

在任何專案目錄中開啟終端機並啟動 Claude Code：

```bash
cd /path/to/your/project
claude
```

您將看到 Claude Code 歡迎畫面，其中包含會話資訊、最近的對話和最新更新。  
輸入 `/help` 查看可用命令，或輸入 `/resume` 繼續之前的對話。

---

## 步驟 4：提出您的第一個問題

如果您剛打開一個專案，可以先讓 Claude 幫您掃描環境。試試這個實用的「起手式」：

```
請幫我分析這個專案：用到了哪些主要技術？程式的進入點在哪裡？
```

Claude 將會自動讀取檔案並為您梳理出摘要。如果您想了解架構，也可以這樣問：

```
請簡單解釋一下這個專案的資料夾結構，如果是核心邏輯應該去哪個資料夾找？
```

除此之外，您也可以直接詢問 Claude 關於它自己的能力：

```
Claude Code 可以幫我做哪些自動化工作？
```

```
我如何在 Claude Code 中建立自訂 skills？
```

> Claude Code 會根據需要讀取您的專案檔案，您不必手動新增上下文。

---

## 步驟 5：進行您的第一次程式碼修改

嘗試一個簡單的任務（具體指出語言與檔案名稱會更好）：

```
請在 main.py 中新增一個 Python 的 hello world 函式
```

Claude Code 將：

1. 找到合適的檔案
2. 向您顯示建議的修改
3. 請求您的批准
4. 進行編輯

> Claude Code 在修改檔案前**一律**請求許可。您可以批准單個修改，或為會話啟用「全部接受」模式。

## 步驟 6：在 Claude Code 中使用 Git

Claude Code 讓 Git 操作變得對話式，您不再需要死記硬背各種複雜的 Git 指令。

**最常用的收尾動作：**
完成修改後，強烈建議您直接使用內建的快捷命令來確認與提交：

**步驟 1：確認修改內容**

輸入 `/diff` 開啟互動式差異檢視器，確認您剛才對 `main.py` 的變更。

**步驟 2：提交修改**

輸入 `/commit`，Claude 會自動：
1. 分析您的修改內容
2. 生成合適的 commit 訊息
3. 執行 `git commit`

**更進階的 Git 操作，您可以直接用白話文交辦：**

```
請幫我建立一個名為 feature/auth 的新分支，並切換過去
```

```
請幫我解決目前的合併衝突，請優先保留我剛剛對 main.py 的修改
```

```
幫我把最後 3 次的 commits 整理合併（rebase）成一個
```

> **💡 避坑小提醒：關於 Git 推送 (Push)**
> 如果您希望 Claude Code 能夠順暢地幫您把程式碼推送到 GitHub/GitLab，請確保您的本地電腦已經設定好 **Git SSH 金鑰**。如果使用 HTTPS 方式連線，終端機可能會卡在要求輸入帳號密碼的畫面，這會導致 Claude Code 的任務被迫中斷。

## 步驟 7：修復錯誤或新增功能

當您熟悉了前面的基本操作後，就可以開始指派真正的開發任務。

**開發新功能**（建議加上明確的驗證要求）：

```
為 main.py 新增使用者註冊表單「密碼長度」與「信箱格式」的輸入驗證。修改完成後，請幫我執行相關的單元測試。
```

**修復現有 Bug**（建議先讓它給方案，不要盲目改）：

```
目前 main.py 的註冊功能有一個 Bug，使用者居然可以提交空表單。請幫我定位問題並給出修復思路，等我確認後再改程式碼。
```

接收到這類指令後，Claude Code 將：

1. 定位相關程式碼並理解上下文
2. 向您解釋原因並給出修復方案（如果您有要求）
3. 實作解決方案
4. 主動執行測試與驗證（如果您有要求）

## 步驟 8：嘗試其他常見工作流程

**重構程式碼**（提醒：重構最怕範圍失控，請務必加上限制）

```
請幫我重構身份驗證模組，改用 async/await。請注意：這次只做最小範圍的重構，不要順手修改其他無關的邏輯，改完後請確保專案仍能順利啟動。
```

**撰寫測試**（提醒：要求覆蓋邊界情況）

```
請針對 main.py 中的註冊與信箱驗證函式撰寫單元測試，請盡可能覆蓋各種極端邊界情況（Edge Cases）。
```

**更新文件**（提醒：讓 Claude 自動分析改動）

```
請根據我剛剛的程式碼改動與新加入的套件，自動幫我把對應的安裝說明與環境變數要求，更新到 README.md 中。
```

**程式碼審查**（提醒：給予特定的審查視角）

```
請從「資安」與「效能」的角度，嚴格審查我剛剛對會員模組的修改，並條列出潛在的風險與改善建議。
```

## 基本命令速查表

| 命令 | 功能 | 範例 |
|------|------|------|
| `claude` | 啟動互動模式 | `claude` |
| `claude "task"` | 執行一次性任務 | `claude "fix the build error"` |
| `claude -p "query"` | 執行一次性查詢後退出 | `claude -p "explain this function"` |
| `claude -c` | 在當前目錄繼續最近的對話 | `claude -c` |
| `claude -r` | 恢復之前的對話 | `claude -r` |
| `claude commit` | 建立 Git 提交 | `claude commit` |
| `/clear` | 清除對話歷史 | `/clear` |
| `/help` | 顯示可用命令 | `/help` |
| `exit` 或 Ctrl+C | 退出 Claude Code | `exit` |

## 新手技巧

### 對您的請求要具體

❌ 不要說：`修復錯誤`

✅ 嘗試：`修復登入錯誤，使用者輸入錯誤憑證後看到空白畫面`

### 使用分步說明

將複雜任務分解為步驟：

```
1. 為使用者設定檔建立新的資料庫資料表
2. 建立 API 端點以取得和更新使用者設定檔
3. 建構允許使用者查看和編輯其資訊的網頁
```

### 讓 Claude 先探索

在進行修改之前，讓 Claude 理解您的程式碼：

```
分析資料庫結構描述
```

```
建構一個管理員儀表板，顯示最近一週內註冊且通過信箱驗證的新使用者統計
```

### 使用快捷鍵節省時間

- 按 `?` 查看所有可用的快捷鍵
- 使用 `Tab` 進行命令補全
- 按 `↑` 查看命令歷史
- 輸入 `/` 查看所有命令和 skills
